# Profile: Small Team

For 2-10 developers working on shared repos. The sweet spot: full contract system
with CI enforcement, but no cross-repo infrastructure yet.

## Recommended Tier: 2 (Adopted)

Set in `.rebarrc`:
```
tier = 2
```

Enforces: contract headers, contract refs, TODOs, doc freshness, registry consistency.

## Key Additions Over Solo

### CI Enforcement

Run `scripts/ci-check.sh` in your CI pipeline. This is the single most valuable
addition at Tier 2 — it catches drift before it compounds:

```yaml
# .github/workflows/rebar.yml
- name: Rebar checks
  run: ./scripts/ci-check.sh
```

### CODEOWNERS for Contracts

```
# .github/CODEOWNERS
architecture/   @team-lead
```

Contract changes need review. This is 5 minutes of setup that prevents accidental contract drift.

### Computed Metrics

METRICS becomes generated, not maintained:
```bash
# CI generates METRICS after tests pass
scripts/check-ground-truth.sh  # or compute-metrics.sh when available
```

### Auto-Generated QUICKCONTEXT Section

Keep the human-written section, auto-generate the metrics:
```markdown
## Current State
<!-- AUTO-GENERATED by CI — do not edit below this line -->
| Metric | Value | Updated |
|--------|-------|---------|
| Tests passing | 847/850 | 2026-03-20 |
<!-- END AUTO-GENERATED -->

## What's Happening (human-written)
Working on offline auth flow. Alex is on the sidebar refactor.
```

### PR Template

```markdown
<!-- .github/pull_request_template.md -->
## Contracts Affected
- [ ] List contracts this PR touches

## Breaking Changes
- [ ] None / describe

## Docs Updated
- [ ] Which docs were updated (or confirm none needed)
```

## When to Level Up

Move to [department.md](department.md) when:
- You have 10+ repos with cross-repo contract dependencies
- New hires can't find which repo owns what
- Breaking contract changes surprise other teams
- You need a catalog for cross-repo discovery

## Setup Time

~45 minutes. Copy everything, customize the Cold Start Quad, set up CI, add CODEOWNERS.
